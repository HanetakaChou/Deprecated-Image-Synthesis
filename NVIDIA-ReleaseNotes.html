NVIDIA Direct3D 11 SDK – Release Notes

Version 11.0, February 2011

sdkfeedback@nvidia.com

 

The NVIDIA Direct3D 11 SDK demonstrates some of the cool, next-generation techniques that are possible with Microsoft’s DirectX 11.    The SDK includes both Direct3D code samples and whitepapers explaining the effects in detail. The latest version of the NVIDIA SDK is available at developer.nvidia.com.

 

This NVIDIA SDK is entirely new for DirectX 11.  It stands alongside the previous NVIDIA SDK 10.0, rather than superceding it.

 

Prerequisites

 

To run the NVIDIA SDK, you require a graphics card which supports DirectX 11, such as the GeForce 400 Series.  

 

Full source is provided.  To compile the samples, you additionally require:

 

  1. Microsoft DirectX SDK from June 2010;
  2. Microsoft Visual Studio 2008;

 

In the absence of these items, the SDK contains short movies which demonstrate the features of each sample.  These can be accessed via the SDK Browser.

 

Getting Started

 

Install the package.  It will place an icon on your desktop: NVIDIA Direct3D SDK 11 Browser.  Click on this to launch the browser.  You can use the browser to examine an overview of each sample.  There are links leading to the executable demos, short videos, whitepapers and the source files.

Compilation Setup

To compile the samples Visual Studio must first be configured with the correct paths for the DirectX SDK.  (Tools menu -> Options; Go to Projects and Solutions category, then VC++ Directories page.)  Settings are required for include directories and libraries.  In a typical installation these are:

 

C:\Program Files\Microsoft DirectX SDK (June 2010)\Include

C:\Program Files\Microsoft DirectX SDK (June 2010)\Lib\x86

 

The DirectX 11 setup program may set these for you.  (The NVIDIA SDK installer cannot set these paths because the details may differ on your system.)

 

There is a top level solution for all samples located at:

 

Visual Studio 2008: Source\Samples_2008.sln

 

(See the above diagram of the directory structure.)

 

Individual projects do not have their own solutions.  You can create individual solutions by opening the vcproj project files.  In this case, dependencies will have to be added: all projects depend upon DXUTCore.vcproj and DXUTOptional.vcproj; many also depend upon D3DXEffects11.vcproj.

 

Debug binaries of the shared libraries are not included. To build a debug version of the samples, first  build debug libraries using the solution provided in “<root>\Source\Common”.

Known Issues

 

The NVIDIA Direct3D 11 SDK uses the DXUT framework library for window management, user interface, etc.  There are known issues.  In particular, many options in the Change device dialog cause crashes.

 

The TerrainTessellation sample may cause a user-mode crash in a 266.58 driver.  This has been fixed in newer driver releases.  Please update your driver to the latest available from www.nvidia.com.
